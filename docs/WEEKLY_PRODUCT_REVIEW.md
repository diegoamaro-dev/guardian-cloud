# Guardian Cloud — Weekly Product Review

## 🎯 Objetivo

Evaluar el estado real del producto cada semana.

Evitar:
- autoengaño
- desviaciones
- acumulación de errores

---

## 🧨 Regla principal

> No avanzar si el sistema actual no funciona en condiciones reales

---

# 🧠 1. Uso real

- ¿Cuántas personas han usado la app esta semana?
- ¿Cuántas sesiones se han creado?

Evaluación:

- [ ] uso real (más de 1 sesión por usuario)
- [ ] uso puntual
- [ ] no uso

---

# 🧠 2. Activación

- ¿El usuario empieza a grabar en <2 segundos?

- [ ] sí
- [ ] no

Si no:

→ problema crítico de UX

---

# 🧠 3. Claridad

¿El usuario entiende sin explicación?

- ¿sabe que está grabando?
- ¿sabe que está protegido?

- [ ] sí
- [ ] dudas
- [ ] no

---

# 🧠 4. Supervivencia

- ¿Las sesiones suben correctamente?

- [ ] >95%
- [ ] 80–95%
- [ ] <80%

Si <95%:

→ problema crítico

---

# 🧠 5. Recovery

- ¿El sistema recupera tras:

  - kill app
  - reinicio
  - background?

- [ ] sí
- [ ] parcialmente
- [ ] no

---

# 🧠 6. Export

- ¿El usuario puede recuperar la evidencia?

- [ ] sí
- [ ] parcial
- [ ] no

---

# 🧠 7. Fricción

¿El usuario ha tenido dudas?

Ejemplos:

- “¿está grabando?”
- “¿se ha guardado?”

- [ ] ninguna
- [ ] algunas
- [ ] muchas

---

# 🧠 8. Errores críticos

Listar:

- pérdida de chunks
- fallo de subida
- export corrupto

---

# 🧠 9. Decisiones de la semana

¿Qué se ha implementado?

¿Ha mejorado:

- [ ] supervivencia
- [ ] claridad
- [ ] confianza

Si no:

→ mala decisión

---

# 🧠 10. Ideas nuevas

Listar ideas generadas:

Para cada una:

- ¿pasa el Feature Evaluation Template?
- ¿está en fase actual?

---

# 🧠 11. Estado del producto

- [ ] listo para avanzar
- [ ] necesita corrección
- [ ] no usable en condiciones reales

---

# 🧨 12. Acción de la semana siguiente

Elegir SOLO UNA prioridad:

- [ ] mejorar UX
- [ ] mejorar supervivencia
- [ ] mejorar export
- [ ] validar con usuarios
- [ ] corregir errores críticos

---

# 🧨 Regla final

> Mejorar lo que duele, no lo que apetece

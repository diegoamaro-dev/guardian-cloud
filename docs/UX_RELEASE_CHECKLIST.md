# Guardian Cloud — UX Release Checklist

## 🎯 Objetivo

Validar que la app es usable bajo estrés antes de cualquier release.

---

## 1. ACTIVACIÓN

* [ ] El usuario puede iniciar grabación en <2 segundos
* [ ] No hay decisiones previas (ej: audio/video)
* [ ] No hay confirmaciones innecesarias
* [ ] El botón principal es obvio sin explicación

---

## 2. FEEDBACK INMEDIATO

Al iniciar:

* [ ] Hay vibración o feedback claro
* [ ] La UI cambia instantáneamente
* [ ] Se muestra "Grabando" sin ambigüedad

---

## 3. CLARIDAD DE ESTADO

Durante grabación:

* [ ] El usuario sabe que está grabando
* [ ] El usuario sabe que se está subiendo
* [ ] El usuario entiende cuándo está protegido

Lenguaje:

* [ ] NO se usan términos técnicos
* [ ] Se usan palabras como:

  * "Grabando"
  * "Subiendo"
  * "Protegido"

---

## 4. ELIMINACIÓN DE FRICCIÓN

* [ ] No hay elementos que distraigan en la HOME
* [ ] No hay información innecesaria durante grabación
* [ ] No hay múltiples acciones principales en pantalla

---

## 5. HISTORIAL

* [ ] El usuario encuentra una sesión en <5 segundos
* [ ] Los estados son claros:

  * Protegido
  * Parcial
  * Error
* [ ] NO se muestran datos técnicos (chunks, hashes)

---

## 6. EXPORT

* [ ] El botón de exportar es visible y claro
* [ ] El usuario puede exportar sin ayuda
* [ ] El archivo resultante es usable fuera de la app

---

## 7. CONFIANZA

* [ ] El usuario siente que la evidencia está protegida
* [ ] No hay dudas durante el uso
* [ ] No hay mensajes ambiguos

---

## 8. TEST CON USUARIOS (OBLIGATORIO)

* [ ] 3 personas prueban la app sin explicación
* [ ] Se les pide: "usa esto si te pasa algo raro"

Verificar:

* [ ] Tiempo de reacción
* [ ] Dudas
* [ ] errores de uso

---

## 9. CRITERIO DE BLOQUEO

NO se lanza release si:

* el usuario duda
* tarda en reaccionar
* no entiende estados
* necesita explicación

---

## 🧨 Regla final

> Si el usuario tiene que pensar, no está listo
